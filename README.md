## Основные настройки

| Переменная              | Значение по умолчанию | Описание |
|-------------------------|-----------------------|----------|
| `PORT`                  | `5500`                | Порт API |

---

## JWT

| Переменная                  | Пример                        | Описание                                           |
|-----------------------------|-------------------------------|----------------------------------------------------|
| `JWT_ACCESS_SECRET_KEY`     | `JWT_ACCESS_SECRET_KEY_01`    | Секретный ключ для генерации **access token**.     |
| `JWT_REFRESH_SECRET_KEY`    | `JWT_REFRESH_SECRET_KEY_02`   | Секретный ключ для генерации **refresh token**.    |
| `JWT_ACCESS_EXPIRES_IN`     | `10m`                         | Время жизни access token (например, `10m`, `1h`).  |
| `JWT_REFRESH_EXPIRES_IN`    | `7d`                          | Время жизни refresh token (например, `7d`, `30d`). |

---

## База данных (MySQL)

| Переменная      | Пример         | Описание                               |
|-----------------|----------------|----------------------------------------|
| `DB_HOST`       | `mysql`        | Хост базы данных в Docker‑сети.        |
| `DB_PORT`       | `3306`         | Порт MySQL внутри сети (по умолчанию)  |
| `DB_USER`       | `admin`        | Пользователь базы данных.              |
| `DB_PASSWORD`   | `adminpass`    | Пароль пользователя базы данных.       |
| `DB_NAME`       | `file_storage` | Имя базы данных.                       |

---

## Безопасность

| Переменная        | Пример | Описание                                                 |
|-------------------|--------|----------------------------------------------------------|
| `BCRYPT_ROUNDS`   | `10`   | Количество раундов хэширования паролей с помощью bcrypt. |

---

## Хранение файлов

| Переменная              | Пример          | Описание                              |
|-------------------------|-----------------|---------------------------------------|
| `FILE_STORAGE_SOURCE`   | `./files_data`  | Путь к директории для хранения файлов |

---

## Запуск

1. Создайте файл `.env.docker` в корне проекта.
2. Заполните его переменными окружения (используйте из примеров).
3. Запуск проекта: `docker compose up -d --build`
